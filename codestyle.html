let and const


function foo() {
  // throw ReferenceError 'a' is not defined
  console.log(a);
  let a = 1;
}
foo();

function bar(a) {
  if (a == 1) {
    let b = 2;
  }

  // throw ReferenceError 'b' is not defined
  console.log(b);
}
bar(1);

JavaScript 常用规范

通用规范

  • 使用 Unix 换行符
  • 使用 utf-8 编码
  • 使用 2 空格缩进
  • 使用具有良好描述性的变量名
  • 一个函数的长度不要超过一屏
  • 每个变量声明对应一个声明符
  • 除了有共识的缩写，例如 tmp，不要使用缩写
  • 每一行的内容，包括空白符，不要超过 100 个字符

  • Guard Clauses
  • 抽离成函数
  • 使用 try catch

如何重构“箭头型”代码

使用 ES6 的方式

  • arrow function
  • class and extends

  • let and const
  • module
  • parameters default value
  • rest
  • shorthand

arrow function


() => {};
a => a;
(a = 10) => a;
[1, 2, 3].map((number, index) => ({
  [index]: number,
}));

// bad
const itemHeight = item => item.height > 256 ? item.largeSize : item.smallSize;
// good
const itemHeight = (item) => {
  const { height, largeSize, smallSize } = item;
  return height > 256 ? largeSize : smallSize;
};

import & export


// bad
import foo from 'foo';
// … some other imports
import { named1, named2 } from 'foo';
// good
import foo, {
  named1,
  named2,
} from 'foo';

// bad
export { es6 as default } from './StyleGuide';
// good
import { es6 } from './StyleGuide';
export default es6;

杂项

  • 始终以分号结尾
  • 使用单引号字符串
  • 尽可能的使用纯函数，不要修改传入参数
  • 使用 map / every / filter / reduce 等高阶函数
  • 给事件传递数据时使用 Hash 的方式
  • 避免使用 `function (arg1, arg2, cb, errCB) {}`

ESLint 及其用法

ESLint 是什么

The pluggable linting utility for JavaScript and JSX

如何使用 ESLint


// 全局安装 eslint
npm i -g eslint

// 初始化 eslint，最好是一个 npm 项目
eslint --init

// 使用 eslint 命令行程序检查代码
eslint yourfile.js

Configuring Rules

  • 'off' or 0
  • 'warn' or 1
  • 'error' or 2
  • ["level", "option value"]

Configuration File

  • YAML - .eslintrc.yml/.eslintrc.yaml
  • JS - .eslintrc.js
  • JSON - .eslintrc.json
  • Deprecated - .eslintrc
  • package.json - eslintConfig

Configuration File - YML


rules:
  eqeqeq: warn,
  curly: error,
  quotes:
    - error
    - double

Configuration File - JSON


{
    "rules": {
        "eqeqeq": "warn",
        "curly": "error",
        "quotes": ["error", "double"]
    }
}
